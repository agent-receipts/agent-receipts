# Security Policy

## Reporting a Vulnerability

The agent-receipts team takes security vulnerabilities seriously. We appreciate your efforts to responsibly disclose any issues you find.

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, please report them through GitHub Security Advisories:

1. Go to the repository's **Security** tab and click **"Report a vulnerability"**, or open the following URL directly: https://github.com/agent-receipts/mcp-proxy/security/advisories/new
2. Fill in the details of the vulnerability
3. Submit the report to create a private security advisory draft that is only visible to the maintainers and you

You can also refer to [GitHub's documentation on privately reporting a security vulnerability](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability).

## What to Include

When reporting a vulnerability, please include:

- A description of the vulnerability
- Steps to reproduce the issue
- The potential impact
- Any suggested fixes (if applicable)

## Response Timeline

- We will acknowledge receipt of your report within **3 business days**.
- We will provide an initial assessment within **10 business days**.
- We will work with you to understand and resolve the issue promptly.

## Supported Versions

Security updates are applied to the latest release. We recommend always running the most recent version.
